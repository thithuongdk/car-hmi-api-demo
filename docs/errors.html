Error Codes Reference

The current API returns errors in two envelopes depending on endpoint family:

// Numeric app code envelope (err)
{
  "error": "Short human-readable message",
  "code": 3002,
  "id": "VAL_MISSING_FIELD"
}

// Detail envelope (apiErr)
{
  "detail": {
    "code": "profile_not_found",
    "message": "Profile 'U0' khong tim thay",
    "profile_name": "U0"
  }
}

Optimistic Locking

For PUT /config, client must send the latest numeric section_id or server returns 409 VAL_CONFLICT. For PUT /api/profile, section_id is optional but if provided and stale, server returns 409 profile_section_mismatch.
