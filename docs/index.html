Modes & Profiles
App hoạt động ở một trong hai mode. Mode ảnh hưởng tới tab nào hiển thị và API nào FE có thể gọi.
ModeHiển thịTabs accessible
User Chỉ signals thuộc active profile Dashboard, Profiles, API Log
Dev Tất cả signals Dashboard, Profiles, Config, Signals Info, API Log
Profile values: Dev, U0, U1, U2, UN — Dev profile là default, hiển thị tất cả signals.
section_id — Optimistic Locking
Cơ chế tránh ghi đè xung đột giữa nhiều client.
⚠ Bắt buộc với mọi request PUT /api/profile và PUT /config.
Hành độngMô tả
Client gửi PUTPhải kèm section_id bằng giá trị hiện tại từ server.
Server matchGhi thành công → trả về data mới với section_id + 1.
Server mismatchTừ chối → 409 Conflict. Client cần GET lại rồi retry.
Tạo mới (POST)section_id do server cấp, client không cần gửi.

Profiles API
Quản lý user profiles — mỗi profile map tới tập hợp signals hiển thị trên Dashboard khi ở User mode.
GET /api/profiles
User
▶

Trả về danh sách tất cả profiles đã lưu.

Response 200
"section_id": 123, "profiles": [ { "profile_name": "U0", "signals": ["EngineSpeed", "CoolantTemp", "HB_FL_ActivationLevel"] }, { "profile_name": "U1", "signals": ["EngineSpeed", "CoolantTemp"] } ]
GET /api/profile?name={profile_name}
User
▶

Lấy thông tin một profile theo tên.

Query Params
name string — Profile name, e.g. "U0"
Response 200
{ "section_id": 123, "profile_name": "U0", "signals": ["EngineSpeed", "CoolantTemp", "HB_FL_ActivationLevel"], "selected": true }
POST /api/profile
User
▶

Tạo mới một profile.

Request Body
{ "profile_name": "U2", "signals": ["EngineSpeed", "HB_FL_ActivationLevel"], "selected": false }
Response 201
{ "section_id": 1, "profile_name": "U2", "signals": ["EngineSpeed", "HB_FL_ActivationLevel"], "selected": false }
PUT /api/profile
User
▶

Cập nhật profile. Yêu cầu section_id khớp với giá trị hiện tại — xem Optimistic Locking.

Request Body
{ "section_id": 123, "profile_name": "U0", "signals": ["EngineSpeed", "CoolantTemp", "HB_FL_ActivationLevel", "HB_FR_ActivationLevel"], "selected": true }
Response 200
{ "section_id": 124, "profile_name": "U0", ... }
Response 409 — section_id mismatch
{ "error": "Conflict: section_id mismatch", "current_section_id": 124 }
DEL /api/profile/{profile_name}
User
▶

Xoá profile theo tên.

Response 204
(no body)
Response 404
{ "error": "Profile not found" }

Signals API
Đọc giá trị signals realtime và ghi các writable signals. Có ở cả User và Dev mode.
GET /signals
User Dev
▶

Snapshot giá trị hiện tại của tất cả signals. Dùng cho initial load; sau đó dùng WebSocket để nhận updates.

Response 200
{ "timestamp": "2024-06-01T12:00:00Z", "signals": [ { "name": "EngineSpeed", "value": 3000 }, { "name": "CoolantTemp", "value": 90 }, { "name": "HB_FL_ActivationLevel", "value": 2 }, { "name": "HB_FR_ActivationLevel", "value": 0 } ] }
GET /signals/available
User Dev
▶

Full metadata của tất cả signals — unit, min, max, writable, states enum. Gọi một lần on startup.

Response 200
{ "signals_info": [ { "name": "EngineSpeed", "unit": "rpm", "min": 0, "max": 8000, "description": "Engine speed in revolutions per minute", "states": [], "values": null }, { "name": "HB_FL_ActivationLevel", "unit": "", "min": 0, "max": 7, "description": "HB FL Activation Level", "states": [ { "value": 0, "description": "Level 1" }, { "value": 1, "description": "Level 2" }, ... (8 states total: Level 1–8 for values 0–7) ], "values": null } ] }
PUT /signals/{signal_name}
User Dev
▶

Ghi giá trị cho một writable signal. BE sẽ forward xuống CAN writer. Chỉ áp dụng cho HB_FL_ActivationLevel, HB_FR_ActivationLevel.

Request Body
{ "timestamp": "2024-06-01T12:00:00Z", "name": "HB_FL_ActivationLevel", "value": 3 }
Response 202 Accepted
{ "status": "queued" }
Response 400 — signal not writable or out of range
{ "error": "Signal is not writable" }
POST /signals/batch_update
User Dev
▶

Ghi đồng thời nhiều writable signals trong một request (sync/bulk write).

Request Body
{ "timestamp": "2024-06-01T12:00:00Z", "updates": [ { "name": "HB_FL_ActivationLevel", "value": 3 }, { "name": "HB_FR_ActivationLevel", "value": 5 } ] }
Response 202 Accepted
{ "status": "queued", "count": 2 }
WebSocket
Realtime signal stream — BE push data theo sampling_rate của active config.
WS ws://<host>/ws/signals
User Dev

Kết nối WebSocket để nhận signal updates theo thời gian thực. Interval giữa các frame = 1000 / sampling_rate ms.

Server → Client message
{ "timestamp": "2024-06-01T12:00:00.123Z", "signals": [ { "name": "EngineSpeed", "value": 3150 }, { "name": "CoolantTemp", "value": 91 }, { "name": "HB_FL_ActivationLevel", "value": 2 }, { "name": "HB_FR_ActivationLevel", "value": 0 } ] }
sampling_rate được lấy từ active config. Thay đổi config → WS reconnect với interval mới.
