WebSocket Endpoint

Realtime CAN signal stream — the backend pushes updates at the sampling_rate of the active config. The frontend only needs one persistent connection to receive all signals.

Signal Alias: Mọi subscribe/unsubscribe đều chấp nhận cả name (tên gốc trong CAN DB) và std_name (tên chuẩn hóa từ data/signal_std_name.json). Payload signal trong mỗi frame luôn trả về cả name + std_name.

Connection Lifecycle

Flow: Connect → get snapshot (optional, use GET /signals) → receive push frames continuously → reconnect on disconnect.
EventFrontend action
onopenSend subscribe message for the desired signals (or "*" for all), update UI badge to "Connected"
onmessageParse JSON, update signal values on the Dashboard
oncloseShow "Reconnecting…", call new WebSocket(...) after 2–5 s
onerrorLog error, trigger reconnect

Server → Client Messages

The backend pushes one JSON frame per interval. Only changed signals (delta) or all signals (full) are sent — depends on implementation.

Subscription Confirmation (subscribed)

Sent by the server immediately after a subscribe or unsubscribe client message, followed by an immediate value snapshot for the new subscription set.

// When subscribed to specific signals: { "type": "subscribed", "signals": ["ARS_FL_InjuryRiskAdaptive", "OMS_FL_HandsOnWheel"], "count": 2 } // When subscribed to all signals: { "type": "subscribed", "signals": "*", "count": 167 }
FieldTypeDescription
type"subscribed"Message type identifier
signalsstring[] | "*"Current active subscription list, or "*" for all
countnumberNumber of signals now being streamed to this client

Client → Server Messages

Clients can control which signals they receive by sending subscribe or unsubscribe messages. Có thể dùng cả name gốc hoặc std_name để chỉ định signal. To write a signal value, use REST PUT /signals/{name} or POST /signals/batch_update.

Subscribe

Tell the server to stream only the listed signals. Send "*" to revert to all signals. The server immediately responds with a subscribed confirmation + value snapshot.

// Subscribe to specific signals ws.send(JSON.stringify({ "type": "subscribe", "signals": ["ARS_FL_InjuryRiskAdaptive", "OMS_FL_HandsOnWheel", "ACR_FL_RetractRequest"] })); // Subscribe using std_name aliases (also accepted) ws.send(JSON.stringify({ "type": "subscribe", "signals": ["OMS_FL_OccupantWeightMean", "OMS_RL1_SeatbeltMisuse"] })); // Subscribe to ALL signals (default) ws.send(JSON.stringify({ "type": "subscribe", "signals": "*" })); // ← Server responds immediately: // { "type": "subscribed", "signals": ["ARS_FL_InjuryRiskAdaptive","OMS_FL_HandsOnWheel","ACR_FL_RetractRequest"], "count": 3 } // { "timestamp": "...", "signals": [{"name":"ARS_FL_InjuryRiskAdaptive","std_name":"ARS_FL_InjuryRiskAdaptive","value":23}, ...] } ← snapshot

Unsubscribe

Remove specific signals from the active subscription without affecting the rest.

ws.send(JSON.stringify({ "type": "unsubscribe", "signals": ["ACR_FL_RetractRequest"] })); // ← { "type": "subscribed", "signals": ["ARS_FL_InjuryRiskAdaptive","OMS_FL_HandsOnWheel"], "count": 2 }

Ping / Keepalive

ws.send(JSON.stringify({ "type": "ping" })); // ← { "type": "pong" }
Message typeRequired fieldsServer response
subscribesignals: string[] | "*"subscribed ack + immediate snapshot

ping—{ "type": "pong" }
Tip: On connect, send subscribe with the active profile's signal list to reduce bandwidth. The HMI demo does this automatically — dev mode subscribes to all signals, user mode subscribes only to the active profile's signals.

Error Handling & Reconnect

⚠ Browser WebSocket has no built-in reconnect. The frontend must implement exponential backoff manually.
function connectWS(retryMs = 2000) { const ws = new WebSocket("wss://car-hmi-api-demo.onrender.com/ws/signals"); ws.onopen = () => { retryMs = 2000; console.log("WS connected"); }; ws.onclose = () => { console.warn(`WS closed, retry in ${retryMs}ms`); setTimeout(() => connectWS(Math.min(retryMs * 2, 30000)), retryMs); }; ws.onerror = (e) => console.error("WS error", e); return ws; }

JavaScript Integration

// 1. On app startup - fetch metadata once const res = await fetch("/signals/available"); const { signals_info } = await res.json(); // signals_info: [{ name, unit, min, max, writable, states, value }, ...] // 2. Connect WebSocket const ws = new WebSocket("wss://car-hmi-api-demo.onrender.com/ws/signals"); ws.onopen = () => { badge.textContent = "● Connected"; }; ws.onmessage = (event) => { const { timestamp, signals } = JSON.parse(event.data); signals.forEach(({ name, value }) => { const el = document.getElementById(`sig-${name}`); if (el) el.textContent = value; }); }; ws.onclose = () => { /* reconnect logic */ }; // 3. Write writable signal (REST, not WS) async function writeSignal(name, value) { await fetch(`/signals/${name}`, { method: "PUT", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ name, value, timestamp: new Date().toISOString() }) }); }
import { useEffect, useRef, useState } from "react"; export function useCANSignals(url = "wss://car-hmi-api-demo.onrender.com/ws/signals") { const [signals, setSignals] = useState({}); const wsRef = useRef(null); useEffect(() => { let retryMs = 2000; let timer; function connect() { const ws = new WebSocket(url); wsRef.current = ws; ws.onmessage = (e) => { const { signals: updates } = JSON.parse(e.data); setSignals(prev => { const next = { ...prev }; updates.forEach(s => { next[s.name] = s.value; }); return next; }); }; ws.onclose = () => { timer = setTimeout(() => { retryMs = Math.min(retryMs*2, 30000); connect(); }, retryMs); }; } connect(); return () => { wsRef.current?.close(); clearTimeout(timer); }; }, [url]); return signals; // { ARS_FL_InjuryRiskAdaptive: 23, OMS_FL_HandsOnWheel: 3, ... } }
